# v1.0.x — LTS (Baseline)

> Primera versión en producción. Usada por el equipo para despliegues de `operativo-api` (Laravel) + `echo-logistics` (Angular) en la org `devtyt` de Bitbucket.

---

## Solicitud

Crear una CLI que genere documentos de despliegue en Confluence automáticamente a partir de un issue key de Jira y un commit hash de Git (backend y/o frontend).

A lo largo del ciclo v1.0.x se fueron agregando:
- v1.0.1–v1.0.7: estabilización inicial, fixes de UX
- v1.0.8–v1.0.11: gestión de múltiples proyectos (`deploy-doc project`), auto-actualización (`deploy-doc update`), `--project` flag en `generate`, config en YAML con `gopkg.in/yaml.v3`, split del installer por plataforma

---

## Motivación

El equipo generaba estos documentos manualmente. Con despliegues frecuentes del proyecto, automatizarlo reducía tiempo y errores. A medida que el equipo creció, surgió la necesidad de soportar múltiples proyectos con distintos repos y rutas, y de mantener el binario actualizado fácilmente.

---

## Diseño técnico

**Self-installer:** el binario se auto-instala al primer run. Divide la lógica de PATH en `path_unix.go` / `path_windows.go`.

**Config YAML:** migración de parser manual `key: value` a `gopkg.in/yaml.v3`. Permite estructuras anidadas (proyectos).

**Proyectos:** `ProjectConfig{BackendPath, BackendRepo, FrontendPath, FrontendRepo}` almacenado como `map[string]*ProjectConfig` en la config. `cfg.GetProject(name)` resuelve: explícito > default > nil.

**workDir en git:** `GetChangedFiles(commitHash, workDir string)` — permite correr `git show` en el directorio del repo correcto sin hacer `cd` previo.

**Auto-update:** `internal/updater` consulta GitHub Releases API para detectar versiones nuevas y hace self-replace del binario.

**Versión embebida:** `internal/build.Version` sobreescrita por ldflags en `make release`.

## Archivos clave

| Archivo | Descripción |
|---|---|
| `main.go` | Entry point con self-installer y background update check |
| `cmd/root.go` | Router + version + help |
| `cmd/init.go` | Configuración interactiva con proyecto opcional |
| `cmd/generate.go` | Flujo principal con soporte `--project` y workDir |
| `cmd/project.go` | Gestión de proyectos (list/add/default/remove) |
| `cmd/update.go` | Auto-actualización |
| `internal/build/version.go` | Variable `Version` para ldflags |
| `internal/config/config.go` | Config + ProjectConfig con YAML |
| `internal/git/git.go` | `GetChangedFiles(hash, workDir)` |
| `internal/updater/updater.go` | CheckLatest + SelfUpdate |

## Cómo verificar

```bash
# Instalar
make install

# Configurar
deploy-doc init

# Verificar versión
deploy-doc version

# Agregar proyecto
deploy-doc project add

# Generar documento con proyecto
deploy-doc generate --issue APP-XXXX --commit-backend <hash> --project <nombre>

# Actualizar binario
deploy-doc update
```
