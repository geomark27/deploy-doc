# v1.1.1 — Patch

> **Fecha:** 2026-04-10
> **Base:** v1.1.0

---

## Solicitud

1. **Renombrar el binario** de `deploy-doc` a `gtt` — comandos más cortos para flujo de trabajo diario.
2. **Flags cortos** para `generate`: `-i`, `-b`, `-f`, `-p`.
3. **Alias de subcomando**: `g` y `gen` como alternativas a `generate`.
4. **Alias `project ls`** como alternativa a `project list`.
5. **Mejora de interfaz**: colores ANSI y pasos `[n/4]` en el output de `generate`.
6. **Fix crítico en `update`**: el comando ofrecía downgrade cuando la versión instalada era mayor a la última release publicada en GitHub.
7. **Fix `make build/install`**: el binario compilado localmente siempre mostraba `dev` como versión.
8. **Eliminar código muerto**: `cmd/version.go` definía una variable `Version` y una función `runVersion` que nunca se llamaban.

---

## Motivación

El comando más usado es `generate`. Con el nombre anterior y flags largos:
```
deploy-doc generate --issue APP-1999 --commit-backend abc1234 --commit-frontend def5678
```

Con los cambios de esta versión:
```
gtt g -i APP-1999 -b abc1234 -f def5678
```

El bug de downgrade se manifestó porque `CheckLatest` comparaba tags con igualdad de string sin verificar si el remoto era realmente mayor. Dado que `v1.1.1` aún no había sido publicada como GitHub Release, la API devolvía `v1.1.0` y el código lo interpretaba como "versión nueva".

## Diseño técnico

### 1. Renombre del binario

Se cambió `BINARY_NAME=deploy-doc` → `BINARY_NAME=gtt` en el Makefile. Esto impacta automáticamente todos los targets de build y release (los assets ahora se llaman `gtt-linux-amd64`, etc.).

Los nombres del binario en `installer.go` y `assetName()` en `updater.go` se actualizaron a `gtt` / `gtt.exe`.

La ruta de configuración `~/.config/deploy-doc/config.yaml` **no cambia** — preserva la config existente de todos los usuarios que ya tenían instalada la herramienta.

El módulo Go (`github.com/geomark27/deploy-doc`) tampoco cambia — es internal y no hay razón de romper eso.

**Compatibilidad hacia atrás en el update:** los binarios anteriores a v1.1.1 usan `assetName()` que retorna `deploy-doc-linux-amd64`. Si el release de v1.1.1 solo tuviera `gtt-linux-amd64`, el `deploy-doc update` de esos usuarios fallaría con HTTP 404. Para evitarlo, el Makefile crea copias con el nombre legacy (`deploy-doc-*`) y las sube junto a los `gtt-*` en **todos** los releases a partir de v1.1.1. Una vez que todos estén en v1.1.1+, este doble asset es redundante pero inofensivo.

**Migración automática deploy-doc → gtt:** `SelfUpdate` detecta si el binario en ejecución se llama `deploy-doc` / `deploy-doc.exe` (`isLegacy`). En ese caso, en lugar de reemplazarse en el mismo lugar, instala el nuevo binario como `gtt` / `gtt.exe` en la misma carpeta y elimina el `deploy-doc` anterior (en Windows lo renombra a `.old` porque no se puede borrar un ejecutable en uso — `CleanOldBinary` lo limpia al siguiente arranque de `gtt`). `SelfUpdate` retorna `(migrated bool, err error)` para que `cmd/update.go` muestre el mensaje correcto según el escenario.

### 2. Flags cortos y aliases de subcomando

`parseFlags` en `cmd/generate.go` ahora normaliza flags cortos a su forma larga antes de parsear:

```go
var shortFlags = map[string]string{
    "-i": "--issue",
    "-b": "--commit-backend",
    "-f": "--commit-frontend",
    "-p": "--project",
}
```

Soporta `-i APP-1999` (espacio) y `-i=APP-1999` (igual). Los flags largos originales siguen funcionando igual.

En `cmd/root.go`, se agregaron `"g"` y `"gen"` al mapa de comandos apuntando a la misma función `runGenerate`. Los tres aliases son intercambiables.

### 3. Alias `project ls`

Se agregó `"ls": runProjectList` al mapa `projectSubcommands` en `cmd/project.go`.

### 4. Colores ANSI y pasos `[n/4]`

Se definieron constantes y helpers en `cmd/root.go` (accesibles por todo el package `cmd`):

```go
const ( clReset, clBold, clRed, clGreen, clYellow, clCyan )

func stepLabel(n, total int, msg string)  // cyan [n/total] msg
func okLine(msg string)                   // indented green ✓
func warnLine(msg string)                 // indented yellow ⚠
func errLine(msg string)                  // indented red ✗
```

El flujo de `runGenerate` se estructuró en 4 pasos explícitos:
- `[1/4]` Buscar issue en Jira
- `[2/4]` Verificar documentos existentes
- `[3/4]` Leer commits (backend y/o frontend)
- `[4/4]` Crear o actualizar en Confluence

La URL resultante se muestra en cyan para fácil identificación visual.

### 5. Fix downgrade en `updater.CheckLatest`

Se agregó la función `isNewer(candidate, base string) bool` que parsea `vMAJOR.MINOR.PATCH` y compara numéricamente. `CheckLatest` solo retorna una versión si es estrictamente mayor que la actual:

```go
if !isNewer(r.TagName, current) {
    return "", nil
}
```

Si el tag de GitHub es menor o igual al instalado (e.g. usuario en `v1.2.0` pero GitHub tiene `v1.1.0`), retorna vacío — nunca downgrade.

### 6. Fix `make build` sin versión

El target `build` del Makefile pasaba `-ldflags "-s -w"` sin `-X build.Version=...`. Ahora usa:

```makefile
VER=$$(git describe --tags --abbrev=0 2>/dev/null || echo "dev"); \
go build -trimpath -ldflags "-s -w -X $(VERSION_PKG).Version=$$VER" ...
```

Así `make build` y `make install` embeben el último tag git. `make build-all` (release) sigue usando el `VER` explícito pasado por `make release`.

### 7. Eliminación de `cmd/version.go`

`runVersion` nunca estaba registrada en el mapa de comandos — el subcomando `version` se manejaba directamente en `cmd/root.go:31` con `build.Version`. La variable `cmd.Version` era un segundo punto de verdad que nunca se sincronizaba con `build.Version`. Archivo eliminado.

## Cómo verificar

```bash
# Compilar y verificar versión embebida
make build
./bin/gtt version
# Esperado: gtt v1.1.0 (o el último tag)

# Flags cortos
make run ARGS='g -i APP-1 -b abc123 --dry-run'
# Equivalente a: generate --issue APP-1 --commit-backend abc123 --dry-run

# Todos los aliases de generate
make run ARGS='g --dry-run --issue APP-1 --commit-backend abc'
make run ARGS='gen --dry-run --issue APP-1 --commit-backend abc'
make run ARGS='generate --dry-run --issue APP-1 --commit-backend abc'

# Alias project ls
make run ARGS='project ls'
# Debe ser idéntico a: project list

# Fix de downgrade — CheckLatest no ofrece versión menor
# (testear solo si hay una versión publicada menor a la actual en GitHub)
make run ARGS='update'
# Si versión actual >= último release de GitHub → "Ya tienes la versión más reciente."

# Verificar que make install embebe versión correcta
make install
gtt version
# No debe mostrar "dev"
```
